# commit 格式

考虑到节约每次 commit 填写描述的时间，因此简化 commit 的格式，暂定如下：

type: desc

可以配置文档 暂时没有配置

说明：

- type用来说明类型，有以下几种：
  - feat：完成新功能的开发
  - fixbug：代码BUG的修复
  - docs：文档
  - ui：UI相关的修改
  - reformat：代码格式修改
  - refactor：代码重构
  - test：测试
  - chore：构建过程或构建工具的变动
- desc用来**简短描述**此次的变动（如修 bug 的则附上 单号：XXX 即可）
